# طبقة المصادقة والصلاحيات (Auth Layer)

تحتوي هذه المجلّدة على المصادقة وإدارة الجلسات وفرض الأدوار والصلاحيات (RBAC)
للأدوار الثلاثة: **مدير**، **محاسب**، **فني**.

## الملفات

- `types.ts` — الأنواع المشتركة: `Role`، `Session`، `Permission`، `AuthError`، وتحويلات أدوار Prisma.
- `password.ts` — تجزئة كلمات المرور والتحقق منها عبر scrypt (دون مكتبات خارجية).
- `auth-service.ts` — `AuthService`: `login`، `isSessionExpired`، `touchSession`، `authorize`.
- `permissions.ts` — مصفوفة صلاحيات كل دور ودالة `authorize` ورسالة عدم التصريح.
- `guard.ts` — حارس الخادم: `requireSession`، `requirePermission`، `withPermission`، و`AuthorizationError`.
- `config.ts` — إعداد Auth.js (NextAuth) بجلسات JWT وحقل الدور.
- `index.ts` — نقطة التصدير الموحّدة (`@/lib/auth`).

معالِج مسار NextAuth موجود في `app/api/auth/[...nextauth]/route.ts`،
وتوسيع أنواع الجلسة في `types/next-auth.d.ts`.

## ملخص القواعد

- **تسجيل الدخول** (المتطلب 1.1): يتحقق من اسم المستخدم وكلمة المرور المجزّأة وينشئ جلسة مرتبطة بالدور.
- **رفض الاعتماد الخاطئ** (المتطلب 1.2): رسالة عامة بالعربية دون كشف السبب.
- **انتهاء الجلسة** (المتطلب 1.5): تنتهي بعد 30 دقيقة من الخمول (`SESSION_IDLE_TIMEOUT_MS`).
- **الصلاحيات** (المتطلبات 1.3, 1.4): المدير يملك كل الصلاحيات؛ المحاسب يملك الفوترة والعملاء
  والموردين والمصروفات والمخزون والتقارير؛ الفني يعرض مهامه المسندة ويحدّث حالتها فقط.
